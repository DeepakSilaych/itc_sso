ITC SSO Documentation

Welcome to the comprehensive guide for integrating with the ITC Single Sign-On (SSO) service.

1. Overview

The ITC SSO service provides a secure, centralized authentication mechanism for IITB students and projects. It allows seamless login across multiple platforms while maintaining robust security standards.

2. Authentication Flow

  1. Register your project with ITC SSO
  2. Obtain your unique Project ID
  3. Implement SSO login endpoint
  4. Verify user data using session key

3. API Endpoints

Endpoint Method Description
/project/<project_id>/ssocall/ GET SSO Login Redirect
/project/getuserdata POST Retrieve User Session Data

4. Code Examples

Python Example (Requests Library)

import requests

# SSO Login URL
login_url = 'https://sso.iitb.ac.in/project/<project_id>/ssocall/'

# User Data Retrieval
user_data_url = 'https://sso.iitb.ac.in/project/getuserdata'
payload = {
    'sessionkey': 'user_session_key'
}

response = requests.post(user_data_url, json=payload)
user_info = response.json()
JavaScript Example (Fetch API)

// SSO Login Redirect
window.location.href = 'https://sso.iitb.ac.in/project/<project_id>/ssocall/';

// User Data Retrieval
fetch('/project/getuserdata', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify({
        sessionkey: 'user_session_key'
    })
})
.then(response => response.json())
.then(userData => console.log(userData));

5. Error Handling

Common Error Responses

  • 400 Bad Request: Missing or invalid parameters
  • 401 Unauthorized: Invalid session or credentials
  • 403 Forbidden: Access denied
  • 404 Not Found: Resource not found
