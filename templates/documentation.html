Getting Started

To use our SSO service, you need to follow these steps:

  1. Register your application in the SSO service portal.
  2. Obtain your API credentials (Client ID and Client Secret).
  3. Integrate the SSO service using the provided API endpoints.

API Endpoints

Here are the primary API endpoints available:

  • POST /api/register/: Register a new user.
  • POST /api/login/: Authenticate a user.
  • POST /api/verify-email/: Verify user email.
  • POST /api/reset-password/: Request a password reset.

Example API Request

POST /api/login/
Content-Type: application/json

{
    "username": "roll_number",
    "password": "user_password"
}

Authentication Flow

The authentication flow consists of the following steps:

  1. User submits their roll number and password.
  2. The system verifies the credentials.
  3. If valid, a session token is generated and returned to the user.
  4. The user can then use this token for subsequent requests to protected endpoints.

Error Handling

When interacting with the SSO service, you may encounter various error responses. Here are common HTTP status codes:

  • 400 Bad Request: The request was invalid.
  • 401 Unauthorized: Authentication failed.
  • 403 Forbidden: Access to the resource is forbidden.
  • 404 Not Found: The requested resource was not found.
  • 500 Internal Server Error: An error occurred on the server.

Example Usage

Below is an example of how to use the SSO service in a Python application using the `requests` library:

import requests

# Login to SSO
url = 'https://your-sso-service/api/login/'
data = {
    'username': 'your_roll_number',
    'password': 'your_password'
}

response = requests.post(url, json=data)

if response.status_code == 200:
    token = response.json().get('token')
    print(f'Logged in successfully. Token: {token}')
else:
    print(f'Error: {response.status_code} - {response.json().get("detail")}')

FAQ

How can I reset my password?

You can request a password reset by calling the POST /api/reset-password/ endpoint with your registered email.

What should I do if I encounter an error?

Refer to the error codes section above. If the problem persists, please contact support.

Can I use this service for multiple applications?

Yes, you can register multiple applications and obtain separate API credentials for each.
